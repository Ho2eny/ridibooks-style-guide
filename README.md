# 코드 리뷰 원칙

- 충분히 좋아야 올바르게 리뷰한 것
- 의견보다 사실을 주장할 것
- 사실에는 출처가 따르고, 의견에는 이유가 따를 것
- 코드 중복은 2번까지 허용
- 구체적일 것
- 필요하다면 "오프라인"으로 토론하되, 해결책은 기록을 남길 것
